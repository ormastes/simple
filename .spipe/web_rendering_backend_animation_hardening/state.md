# Feature: web-rendering-backend-animation-hardening

## Raw Request

harden webbrowser, js engine, wasm engine, simple engine, share backend. webgpu hardeb. simple 2d over webgpu harden. harden web rendering and animation.

## Task Type

feature

## Refined Goal

Harden the shared Simple Web rendering backend path so browser, JavaScript/WASM-hosted Simple, WebGPU, Simple2D, and animation surfaces have explicit fail-closed evidence instead of relying on broad parity reports alone.

## Acceptance Criteria

- AC-1: The existing Simple Web production hardening state remains the authority for security and external-host proof gates, and this lane does not overwrite unrelated dirty work.
- AC-2: A local backend/animation hardening slice is documented with concrete scope, excluded external-host gates, and focused verification evidence.
- AC-3: At least one focused executable spec or wrapper covers a shared web rendering backend behavior with real assertions and no placeholder passes.
- AC-4: Focused verification for the selected slice passes or records a concrete blocker with the exact command and failure mode.

## Scope Exclusions

- No claim that macOS Metal, AMD ROCm/HIP, Windows DirectX, or real browser WebGPU device readback is complete from this Linux host.
- No broad release or version bump.
- No cleanup of unrelated dirty files or concurrent lanes.
