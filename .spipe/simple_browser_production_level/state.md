# Feature: simple-browser-production-level

## Raw Request
$sp_dev make simpel browser production level, whole features. include security related feature which not shown on ui. analysis and plan and  impl them with agent teams

## Task Type
feature

## Refined Goal
Make the Simple browser production-ready across visible browser behavior, hidden security boundaries, renderer/platform evidence, and operational verification without relying on UI-only signals.

## Acceptance Criteria
- AC-1: Existing Simple browser production-hardening requirements, architecture, design, system plan, executable specs, generated manuals, and reports are audited against current source and classified as complete, partial, stale, or missing.
- AC-2: Hidden security controls have explicit production requirements and tests for token lifecycle, fail-closed authorization, origin/CSRF posture, secret handling, request parsing, WebSocket safety, response headers, logging redaction, rate limiting, and default production configuration.
- AC-3: Visible browser behavior has explicit production requirements and tests for navigation, renderer correctness, session/restore behavior, error/failure surfaces, keyboard/accessibility basics, and user-facing degraded-mode behavior.
- AC-4: Renderer, GPU, and platform-readback production gates distinguish locally verifiable Linux evidence from external-host evidence for Metal, ROCm/HIP, DirectX, and real browser WebGPU `device_readback`.
- AC-5: Implementation closes every locally actionable production gap discovered by the audits without weakening existing browser hardening behavior or external-host closure rules.
- AC-6: SPipe/SSpec coverage uses concrete assertions and generated manuals; no placeholder passes, boolean-wrapper assertions, or executable `.spl` specs under `doc/06_spec`.
- AC-7: Focused verification commands for changed browser/security/renderer surfaces pass once, and remaining external-host gates are documented as blockers with exact runbook and manifest requirements.

## Scope Exclusions
- Native macOS Metal, AMD ROCm/HIP, Windows DirectX, and real browser WebGPU `device_readback` proof cannot be produced on this Linux host and must remain external-host evidence unless capable hardware is made available.
- Unrelated dirty worktree lanes are not part of this feature unless explicitly integrated later.
