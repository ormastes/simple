## Task Type
feature

## Refined Goal
Make RV32 and RV64 SimpleOS FPGA support production-ready with repeatable local, RTL, QEMU, and physical-board evidence.

## Acceptance Criteria
- AC-1: RV32 and RV64 RTL smoke gates report both lanes in one run.
- AC-2: RV32 and RV64 SimpleOS build/run evidence exists and is current.
- AC-3: FPGA preflight reports tool, board, artifact, JTAG, UART, and bitstream status.
- AC-4: Physical FPGA boot is not marked done until a board bitstream, load path, UART marker, and SimpleOS payload all pass.
- AC-5: Current blockers are recorded in `doc/09_report/`.
- AC-6: RV32 and RV64 authoritative CPU/SoC RTL is generated by the Simple
  compiler from tracked `.spl` hardware roots; empty, stub/fallback,
  emitted-string, copied, or external CPU RTL cannot satisfy this criterion.
- AC-7: RV32 instruction fetch and load/store use production Sv32 translation
  plus PMP enforcement; RV64 uses production Sv39 plus PMP, with executable
  page/access-fault, SATP, SFENCE.VMA, and privilege/trap evidence.
- AC-8: Each declared RV32/RV64 ISA/privilege profile passes focused production
  tests, ACT4 architectural tests, and non-vacuous RVFI/SymbiYosys checks.
- AC-9: The same pinned OpenSBI/Linux/DT/rootfs artifacts boot on QEMU as a
  software oracle and on generated RTL as separate evidence rows.
- AC-10: RV32 and RV64 generated RTL each reach `login:`, accept terminal input,
  reach a shell, execute `ls /`, and retain DUT-origin transcripts and hashes.
- AC-11: The connected FPGA is programmed independently for RV32 and RV64;
  each row retains board identity, timing/resources, bitstream/programming
  hashes, and a board-origin bidirectional Linux login/`ls` transcript.
- AC-12: The final guide and mirrored SPipe manual cover `.spl` hardware source,
  compiler diagnostics/source maps, simulation, formal, synthesis, bitstream,
  programming, and terminal proof, and receive normal/highest-capability review.

## Scope Exclusions
Do not invent a second RISC-V core, bootloader, or FPGA framework. Reuse the existing RV32/RV64 RTL, SimpleOS QEMU, and Kria/K26 preflight scripts.
