# TRACE32 Skill - Simple Language

## When to Use This Skill

Use this skill when working with Lauterbach TRACE32 in Simple:
- Configuring TRACE32 for headless/automated operation
- Writing `t32_native` RCL (Remote Control) code
- Writing `t32_gdb` bridge code (GDB RSP via TRACE32)
- Creating PRACTICE (.cmm) startup scripts
- Debugging T32 connectivity (USB probe, RCL port, GDB server)
- Working with STM32 targets via TRACE32

---

## Architecture

```
Simple Code  →  t32rem / RCL API (UDP)  →  PowerView  →  JTAG Probe  →  Target MCU
                                               ↓
                                      GDB Back-End (TCP)  →  GDB Client
```

### Two Remote Interfaces

| Interface | Protocol | Port | Use Case |
|-----------|----------|------|----------|
| `t32_native` (RCL) | UDP NETASSIST | 20000 | Control PowerView directly |
| `t32_gdb` (RSP) | TCP | 30000 | GDB debugging through TRACE32 |

- **RCL** is for controlling TRACE32 itself (run scripts, read registers, manage windows)
- **GDB** is for debugging the target through TRACE32's GDB stub
- The `t32_gdb` path still needs RCL to configure the GDB server inside PowerView

## Host Setup

### Prerequisites

Run once per host:

```bash
scripts/t32_host_setup.shs
```

This script:
1. Installs Lauterbach udev rule → creates `/dev/lauterbach/trace32/<dev>` symlinks
2. Enables PCF bitmap fonts (removes `70-no-bitmaps-except-emoji.conf`, adds T32 fonts)
3. Installs Xvfb + xdotool for headless X11
4. Triggers USB re-enumeration for the probe

### Manual Steps If Needed

```bash
# Fix udev (creates /dev/lauterbach/trace32/ symlinks)
sudo cp /opt/t32/bin/pc_linux64/udev.conf/kernel_starting_2.6.32/10-lauterbach.rules \
        /etc/udev/rules.d/10-lauterbach.rules
sudo udevadm control --reload-rules
sudo udevadm trigger --action=add /sys/bus/usb/devices/<DEVNAME>

# Fix fonts (T32 needs PCF bitmap fonts)
sudo rm /etc/fonts/conf.d/70-no-bitmaps-except-emoji.conf
sudo fc-cache -f

# Reset stuck probe firmware
usbreset "0897:0002"
# If that fails: physically unplug/replug the USB cable
```

## Configuration

### Headless Config (`config/t32/t32_stm_headless.t32`)

```text
OS=
SYS=/opt/t32
TMP=/tmp

PBI=
USB

SCREEN=
INVISIBLE

RCL=
NETASSIST
PORT=20000
```

**Config format rules:**
- Sections separated by blank lines
- Key `KEY=` on its own line, value on next line (like `PBI=\nUSB`)
- `SCREEN=INVISIBLE` requires Xvfb (not truly headless — needs an X display)
- `SCREEN=OFF` is NOT supported in the installed T32 build
- This T32 build supports `NETASSIST` only (NOT `NETTCP` — verified via binary strings)

### Launching T32

```bash
# Start T32 headless (Xvfb provides the X display)
xvfb-run -a scripts/t32_start_stm.shs stm32wb native

# Or manually
xvfb-run -a /opt/t32/bin/pc_linux64/t32marm -c config/t32/t32_stm_headless.t32

# Verify RCL
/opt/t32/bin/pc_linux64/t32rem localhost port=20000 PING
```

## t32rem Commands

`t32rem` uses UDP NETASSIST protocol (1024-byte packets).

```bash
# Basic
t32rem localhost port=20000 PING                    # Health check
t32rem localhost port=20000 "SYStem.Up"             # Connect to target
t32rem localhost port=20000 "SYStem.Down"           # Disconnect target
t32rem localhost port=20000 "QUIT"                  # Close PowerView

# Register access
t32rem localhost port=20000 "Register.Get PC"       # Read PC
t32rem localhost port=20000 "Register.Set PC 0x8000000"

# Memory access
t32rem localhost port=20000 "Data.dump 0x0--0xFF"   # Dump memory
t32rem localhost port=20000 "Data.Set 0x20000000 %Long 0x12345678"

# Execution control
t32rem localhost port=20000 "Break"                 # Halt target
t32rem localhost port=20000 "Go"                    # Resume
t32rem localhost port=20000 "Step"                  # Single step

# Script execution
t32rem localhost port=20000 "DO /path/to/script.cmm"

# GDB server enable (for t32_gdb path)
t32rem localhost port=20000 "SETUP.API.GDB.Enable /PORT 30000"
```

## PRACTICE Scripts (.cmm)

### STM32WB Native Startup (`config/t32/stm32wb_native_start.cmm`)

```cmm
; Load vendor startup, enable semihosting
DO /opt/t32/stm32wb_startup.cmm
SYStem.Option SemiHost ON
Break
ENDDO
```

### GDB Server Startup (`config/t32/stm32wb_gdb_start.cmm`)

```cmm
DO /opt/t32/stm32wb_startup.cmm
SYStem.Option SemiHost ON
SETUP.API.GDB.Enable /PORT 30000
Break
ENDDO
```

## Simple API

### t32_native — RCL Client

**Location:** `src/lib/nogc_sync_mut/debug/remote/protocol/trace32.spl`

```simple
use std.debug.remote.protocol.trace32.*

# Connect to TRACE32 via RCL
val client = Trace32Client(host: "localhost", port: 20000)
client.connect()?

# Basic operations
client.ping()?
client.halt()?                    # Break
client.resume()?                  # Go
client.step()?                    # Single step

# Register access
val pc = client.read_register("PC")?
client.write_register("PC", 0x8000000)?

# Execute PRACTICE command
client.execute("DO startup.cmm")?

client.disconnect()
```

### t32_gdb — GDB Bridge

**Location:** `src/lib/nogc_sync_mut/debug/remote/protocol/t32_gdb_bridge.spl`

```simple
use std.debug.remote.protocol.t32_gdb_bridge.*

# Two-phase connection:
# 1. Use RCL to configure T32 and enable GDB server
val t32 = Trace32Client(host: "localhost", port: 20000)
t32.execute("SETUP.API.GDB.Enable /PORT 30000")?

# 2. Connect GDB to T32's GDB stub
val gdb = GdbClient(host: "localhost", port: 30000)
gdb.connect()?
gdb.read_registers()?
```

### DAP Adapter

**Location:** `src/lib/nogc_sync_mut/dap/adapter/trace32.spl`

```simple
# Trace32Adapter implements the DAP trait for TRACE32-backed debugging
val adapter = Trace32Adapter(host: "localhost", port: 20000)
adapter.initialize()?
adapter.launch(program: "firmware.elf")?
adapter.set_breakpoint(file: "main.c", line: 42)?
adapter.continue()?
```

## Troubleshooting

### "error initializing TRACE32" from t32rem
- PowerView is not running, or RCL port not open
- Check: `ss -ulnp | grep 20000`

### "FATAL ERROR from X-windows; XFT available, but not working with PCF bitmap fonts"
- Fix: `sudo rm /etc/fonts/conf.d/70-no-bitmaps-except-emoji.conf && sudo fc-cache -f`

### "FATAL ERROR from PODBUS-driver: TRACE32 communication error (4,0,0)"
- Probe firmware stuck. Fix: physically unplug/replug USB cable, then `usbreset "0897:0002"`

### "Couldn't claim interface: No such file or directory"
- USB interface not enumerated. Fix: `usbreset "0897:0002"`, check `/sys/bus/usb/devices/<dev>:1.0` exists

### No `/dev/lauterbach/trace32/`
- udev rule missing. Fix: `scripts/t32_host_setup.shs`
