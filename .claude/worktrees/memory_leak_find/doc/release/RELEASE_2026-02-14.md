## Migration Guide

### For Existing Code

#### 1. Update Import Syntax (if using old style)
```simple
# OLD (deprecated)
import app.mcp.prompts

# NEW (recommended)
use app.mcp.prompts.{PromptManager}
```

#### 2. Use Lazy Initialization for FFI (if needed)
```simple
# Pattern for module-level FFI calls
var _cache: T? = nil
fn get_value() -> T:
    if val cached = _cache: return cached
    val result = expensive_ffi_call()
    _cache = Some(result)
    result
```

#### 3. Extern Function Declarations
```simple
# Declare all FFI functions
extern fn rt_function_name(arg: type) -> return_type
```

#### 4. Replace Result<T,E> with Tuples (in interpreter code)
```simple
# Instead of: Result<T, E>
fn parse_version(s: text) -> (bool, Version?, text):
    if error:
        return (false, nil, "error message")
    (true, Some(version), "")
```

## Known Issues & Workarounds

### Parser Generic Field Access
**Issue:** Reserved keyword "tensor" causes parser errors when used as generic parameter name.

**Workaround:** Rename "tensor" variables to "t", "x", or other names.

**Example:**
```simple
# Problematic
fn get<T>(tensor: Box<T>) -> T: tensor.value

# Fixed
fn get<T>(t: Box<T>) -> T: t.value
```

**Status:** Workaround applied to 29 files, all tests passing. Permanent fix requires Rust runtime changes.
