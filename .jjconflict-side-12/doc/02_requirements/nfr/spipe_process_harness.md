# SPipe Process Harness NFR

- NFR-001: Hook path must be fast and append-only.
- NFR-002: Shared logic must be testable without invoking provider CLIs.
- NFR-003: Prevention failures must return a non-zero hook-compatible exit code.
- NFR-004: HUD output must remain a single compact CLI line.
- NFR-005: Existing unrelated worktree changes must not be included in the SPipe commit.
