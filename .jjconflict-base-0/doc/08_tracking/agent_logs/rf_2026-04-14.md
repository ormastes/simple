# Agent RF Log — Registry Full Recurrence — 2026-04-14

## Emission Site

`src/os/services/launcher/launcher.spl:390`
```
serial_println("[launcher] Registry full ({MAX_APPS} slots)")
```
Fires when `app_count >= MAX_APPS` (32) inside `launcher_register()`.

## Call Chain

`shell.init()` / `launch_app()` / `launch_app_baremetal()`
  → `launcher_init()` — registers 4 hardcoded apps, resets `app_count = 0`
  → `_sync_launcher_registry_with_manifests()` — **was supposed to be no-op**

## Root Cause

Squash commit `326cca5b94` (`feat(simpleos-gui): unified small-complete-gui work —
30+ agent commits squashed`) overwrote Agent SF Round 12's `pass_do_nothing` body
with an active loop:

```spl
fn _sync_launcher_registry_with_manifests():
    val manifests = load_app_manifests()   # returns 29 entries
    for manifest in manifests:
        launcher_register(...)             # 25 new unique slots added
```

`default_manifests()` in `src/os/desktop/app_manifest.spl` returns 29 AppManifest
entries. After `launcher_init()` registers 4 hardcoded apps, the loop adds 25 more
unique names = 29 total slots. This is under the 32-slot cap for a single run.

**However**, in interpreter mode the module-level global `app_count` persists
between test iterations without a `launcher_init()` reset. On a second test
iteration the registry starts at 29, the loop tries to add 25 more, and hits
`app_count >= 32` at slot 32, emitting "Registry full (32 slots)". Additionally,
the 25 stub manifest binaries (e.g. `/sys/apps/calculator`) do not exist on the
baremetal image, making this registration semantically incorrect regardless.

The squash also introduced `_manifest_shortcut_key` / `_manifest_shortcut_mod`
helpers that only handle the original 4 names; all other manifests get key=0/mod=0.

## Regression Commit

`326cca5b94` — 2026-04-13 — squash of 30+ agent commits including VT4/B1/HW/QW work

## Fix Applied

Reverted `_sync_launcher_registry_with_manifests` to `pass_do_nothing` in
`src/os/desktop/shell.spl` lines 1013–1023. The 4 hardcoded apps registered by
`launcher_init()` are the correct runtime set. The 29-entry manifest list is for
UI/spotlight/category browsing only, not for launcher registration.

Helper functions `_manifest_shortcut_key` and `_manifest_shortcut_mod` are now
dead code (no callers) but are left in place as they are in `shell.spl` (allowed
file) and may be used by future manifest-backed shortcut work.

## Files Modified

- `src/os/desktop/shell.spl` — `_sync_launcher_registry_with_manifests` body
  reverted to `pass_do_nothing`
