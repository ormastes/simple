# Daily Debug Reports

This directory holds the per-day digests produced by the company bug-report
pipeline (issue #10). One file per day, named `YYYY-MM-DD.md`.

The digests are generated by:

- `/spipe_loop --daily-debug` (orchestrator), which calls
- `bin/itf daily-debug` → `src/app/itf/cmd_daily_debug.spl::run_daily_debug`,
  which is also the entry point used by `/company_bug_report ingest`.

The watermark that bounds each run lives at
`~/.config/itf/spipe_daily.sdn` (mode 0600).

## File Format

Each daily file follows this skeleton:

```markdown
# Daily Debug Digest — 2026-04-26

## Summary

| Metric | Value |
|--------|-------|
| Messages scanned | 7 |
| Artifacts triaged | 12 |

## Per-bug Sections

### [ABC-123] Crash on boot for ZX1 SoC

- Received: 2026-04-26T08:14:00Z
- Jira keys: ABC-123
- MinIO URLs: 3
  - fw : https://minio.example.com/bug/ABC-123/fw.bin
  - dump: https://minio.example.com/bug/ABC-123/core.dmp
  - note: https://minio.example.com/bug/ABC-123/repro.log
- Severity: S1
- Owner: alice@example.com
- Target SoC: ZX1

### [DEF-9] Boot loop on stress test

- Received: 2026-04-26T11:02:00Z
- Jira keys: DEF-9
- MinIO URLs: 1
  - fw : https://minio.example.com/bug/DEF-9/fw.elf
- Severity: S2
- Owner: bob@example.com
- Target SoC: ZY7
```

## Triage Classifier

The driver classifies every MinIO URL it finds by file extension:

| Kind | Extensions |
|------|-----------|
| `fw` | `.bin`, `.elf`, `.hex` |
| `dump` | `.dmp`, `.core`, `.crash` |
| `note` | `.log`, `.txt`, `.md` |
| `unknown` | anything else |

## Notes

- The digest is regenerated each run; older days are preserved verbatim.
- `--dry-run` prints the plan only and writes nothing.
- `--quiet` skips the notify step (step 7) but still writes the digest and
  watermark.
