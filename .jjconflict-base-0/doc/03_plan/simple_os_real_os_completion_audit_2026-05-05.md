# SimpleOS Real-OS Goal Completion Audit (2026-05-05)

Objective audited:

> make simple os be real os run program from file system, port clang, rust to launch from fs. make it more pure simple remove duplication and runnable x86,riscv,arm 32 and 64. check with qemu. and check ssh server, wm, and browser of simple.

## Completion Audit Result

This document is complete as an evidence audit for the objective above. The
objective itself is not fully complete: current evidence proves multi-arch QEMU
smokes, filesystem-backed app/package reads, bounded Clang/Rust filesystem
launch probes, SSH success/rejection probes, WM smoke, browser pixel smoke, and
several purity/duplication cleanups. It does not prove a working in-guest
Clang/Rust compiler frontend, compile output, full RFC SSH behavior, full
browser conformance, or broad duplicate removal.

## Current Missing Work

1. Full `clang` and `rust` compiler execution in guest remains open. Current evidence proves FAT32 VFS executable reads, SMF executable-envelope validation, embedded ELF marker validation, real manifest/proof file reads, launch fixtures/manifests, source/project-input proof markers, filesystem-exec spawn bridge calls in both the dedicated toolchain probe and desktop disk/UEFI lanes, and process-backed app markers for the toolchain apps, not a working in-guest compile. `bin/simple run src/os/port/deploy_toolchains.spl -- --status` currently reports `llvm-cross NOT BUILT`, `compiler-rt NOT BUILT`, `rust-examples NOT BUILT`, `clang-static-guest NOT BUILT`, `rustc-static-guest NOT BUILT`, and `toolchain-disk-bake DISABLED`, so the real compiler payload path is objectively absent. The disk bake marker is now strict: if `build/os/.bake_include_toolchain` exists, the bake fails unless both guest static compiler payloads are present. Tracking bug: `doc/bugs/simpleos_in_guest_toolchain_execution.md`.
2. Continue the duplication/purity follow-up from `doc/03_plan/simple_os_purity_duplication_audit_2026-05-05.md`, especially splitting large syscall/QEMU runner modules, reducing remaining VFS alias duplication, and fixing bounded duplicate-check output.
3. If full RFC SSH is required, replace the bounded bidirectional live success/rejection auth/session probe with real SSH authentication and channel/session verification.
4. Keep the freestanding SMF parser covered by QEMU lanes. A baremetal-only failure in the runtime byte accessor caused FAT32-backed SMF packages to be read successfully but rejected by the parser; `src/os/kernel/loader/smf.spl` now indexes byte arrays directly, and the toolchain VFS probe plus desktop disk/UEFI lanes cover that path.
