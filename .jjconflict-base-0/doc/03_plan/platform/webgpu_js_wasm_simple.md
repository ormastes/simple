# WebGPU JS/WASM/Simple System Test Plan

## Scope

This plan verifies the current WebGPU browser slice across three consumer paths:

- JavaScript page APIs: `window.isSecureContext`, `navigator.gpu`, `navigator.gpu.requestAdapter`, and canvas `webgpu` context exposure.
- Simple script APIs: direct Simple imports of WebGPU context/resources/commands plus Simple 2D and Simple 3D canvas wrappers.
- WebAssembly path: Simple-to-WASM backend/glue correctness, BrowserSession WASM asset instantiation, JS WebGPU global access in the same session, and bounded direct WASM import calls into declared WebGPU host bindings.

Out of scope until implementation exists: full WebGPU CTS, WebGL compatibility, Three.js/Babylon integration, and a complete WASM-originated WebGPU ABI beyond the declared host-import call contract. Real Chrome/Electron WebGPU draw and compute probing is now covered by host-adaptive evidence wrappers: hosts with non-fallback WebGPU produce adapter/device/pipeline/draw-or-dispatch/readback evidence, while hosts without it must return `host-unavailable:*` without substituting software replay pixels or software compute output.

## Pass/Fail Criteria

PASS for the current slice requires REQ-WGPU-001 through REQ-WGPU-008 to pass with real assertions and no `pass_todo`/placeholder assertions. Simple 2D and Simple 3D facades must expose command summaries and WebGPU submission evidence; Simple 3D evidence proves encoded scene upload plus a bounded draw submission, not full semantic rasterization of arbitrary 3D geometry. The browser promise chain must reach WASM instantiation metadata and WebGPU global metadata, `requestAdapter()` must resolve deterministic software-adapter metadata in the same session, nested returned-Promise assimilation for WebGPU callbacks must be covered, and bounded declared WASM-to-WebGPU host import calls for adapter, dispatch-shaped, and buffer-upload-shaped bindings must be covered. Dispatch-shaped bindings must prove WASM-originated `x/y/z` workgroup arguments for both `i32` and void import result shapes; buffer-upload-shaped bindings must prove WASM-originated memory bytes, pointer/length handoff, a Simple2D rectangle payload, and exported `WebAssembly.Memory` buffer visibility to the host callback. WebGPU compute evidence must keep the browser target source-only and host-adaptive. A complete WebGPU host binding surface remains documented later bridge work.

## Risk Areas

- Interpreter mode may load specs without exercising every `it` body in some runner paths; use native/full execution where available before release.
- BrowserSession now proves JS-mediated WASM fetch, `arrayBuffer`, Promise chaining, instantiation, same-session WebGPU global/adapter metadata, `adapter.requestDevice` shape, nested returned-Promise assimilation for WebGPU callbacks, and bounded declared `webgpu.requestAdapter`, `webgpu.dispatch`, and `webgpu.writeBuffer` host import callbacks with WASM-originated workgroup dimensions, memory payload bytes, Simple2D rectangle payload bytes, and exported `WebAssembly.Memory` buffer reads. It now also proves bounded software `device.createBuffer` plus `device.queue.writeBuffer` mutation of observable WebGPU buffer bytes/counts/checksums, a WASM exported-memory `Uint8Array` upload into a software WebGPU buffer, browser-shaped compute encoder/pipeline/pass/submit counters with multi-pass accumulation and invalid command-buffer filtering, WASM-originated dispatch dimensions forwarded into a compute pass, and WASM-originated Simple2D fill byte payload driving a `simple_2d_fill_u32` compute submit. It does not prove a complete WebGPU host binding ABI or driver-backed execution.
- WASM tests prove backend/glue generation, browser-side instantiation, and bounded import invocation, not real hardware WebGPU execution.
- Current WebGPU executor is deterministic software simulation, not a CTS-compliant renderer. `browser_webgpu_chrome_draw_evidence_spec.spl` and `browser_webgpu_chrome_compute_evidence_spec.spl` are the canonical Chrome/Electron probes for real browser WebGPU draw/compute evidence and must report non-success status when the local host cannot expose non-fallback WebGPU or cannot capture/read back expected data. The draw probe now consumes a bounded WASM Simple3D triangle payload string and carries payload byte/checksum provenance; the compute probe accepts compiler-emitted WGSL source and entry metadata, and the WASM Simple2D fill wrapper carries payload byte-count/checksum provenance before dispatching `simple_2d_fill_u32`.
