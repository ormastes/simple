## How goldens update

Each statistics capture above pins a `(mean, stddev, p95, N)` tuple pulled from
a prior verified run; a scenario's default mode only *compares* against that
tuple and never writes to it. Running the same spec with
`--update-golden` re-executes the measurement, recomputes the tuple, and
overwrites the golden file in place — it does **not** re-run the tolerance
check, so a regression can be silently baked in if the flag is used carelessly.
For that reason `--update-golden` runs are tagged `pending-review` in
`test_db.sdn` and excluded from `bin/simple build check` gating until a human
reviews the diff (old tuple vs new tuple, plus the sparkline) and promotes it
via `bin/simple bug-add --approve-golden=<run-id>`. Baselines never move on a
routine green run — only on an explicit, reviewed re-baseline.
