# Architecture: Simple Theme System

## Source Of Truth

`config/themes/theme.sdn` is the theme registry source of truth. It declares the default theme, family packages, concrete theme packages, compatibility aliases, required widget CSS roles, and required icon roles. The default is `aetheric_dark`; legacy names such as `glass_obsidian_dark`, `glass_dark`, and `obsidian_dark` resolve deterministically through the alias table.

Old `.simple-theme` files are compatibility/import inputs only. Runtime GUI, Web WM, hosted WM, BrowserBackend, and Engine2D WM code consume resolved folder packages.

## Ownership

`common.ui.theme_package` owns:

- Registry parsing and alias resolution.
- Family/theme package resolution.
- Validation for missing files, broken references, widget coverage, icon coverage, token layer rules, and unresolved `var(--token)` references.
- Process cache and startup/load-time fingerprints.
- Shared structs for resolved CSS, token maps, widget CSS maps, icons, numeric colors, `UITheme`, `GlassDesignTokens`, and compositor glass config values.

GUI, Web, WM, and Engine2D modules are adapters. They must not parse theme files or copy visual constants when a resolved package field exists.

## Runtime Flow

```text
theme.sdn registry
  -> family package
  -> concrete theme package
  -> ResolvedThemePackage
  -> GUI CSS, Chromium/Electron CSS, Simple Web CSS, BrowserBackend colors,
     hosted WM glass config, Engine2D WM numeric colors, icon defaults
```

CSS composition order is:

1. Family shape CSS.
2. Family widget defaults and per-widget family CSS.
3. Theme base CSS.
4. Theme widget overrides.
5. Icon custom properties.
6. Minimal structural runtime CSS generated by renderer adapters.

## Renderer Adapters

`app.ui.web.html.generate_css(theme)` is the shared Chromium/Electron and Simple Web CSS entrypoint. Electron WM app windows call the same themed Simple Web wrapper used by the pure Simple Web renderer, so app chrome and content inherit one package CSS path.

The Electron host shell has a separate adapter-level theme bootstrap for
macOS/native host WM chrome. `tools/electron-shell/preload.js` reads the
registry default package once and passes composed CSS to
`tools/electron-shell/index.html`, which applies it before host WM rendering.
This path themes Electron-owned `BrowserWindow` shell chrome and taskbar
surfaces directly; it does not make the SimpleWeb renderer responsible for
host-native WM appearance.

`os.compositor.browser_backend.BrowserBackend` resolves numeric colors and package CSS at backend construction. Rendering uses cached fields, sets theme metadata on the DOM root, and does not reread theme files on the hot path.

Hosted WM and Engine2D WM entrypoints load the registry default and pass resolved numeric/glass values into compositor, splash, taskbar, dock, and window chrome drawing paths.

## Cache Policy

`load_theme_package(id)` resolves aliases once per call, caches by canonical id, and records source paths plus a startup fingerprint. Render paths receive `ResolvedThemePackage` fields or cached helper values. Explicit reload may refresh package data; render loops must not parse CSS or scan theme folders.

## Failure Policy

An invalid default theme is a startup/verification failure. Validation reports exact file/key/path messages for missing registry entries, missing family/theme files, broken aliases, missing widgets, missing icons, undefined token references, and token definitions in the wrong layer.
