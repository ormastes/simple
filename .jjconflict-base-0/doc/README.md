# Simple Language — Documentation Wiki

All documentation lives under numbered top-level folders.
`test/` and `todo/` are system-data directories (live `.sdn` databases) and are excluded from this index.

---

## Folder Map

| Folder | Purpose |
|--------|---------|
| [00_llm_process](00_llm_process/) | LLM pipeline config, feature/layer experts, skill commands, templates |
| [01_research](01_research/) | Domain research, language comparisons, local and general references |
| [02_requirements](02_requirements/) | Feature requirements and non-functional requirements |
| [03_plan](03_plan/) | Implementation plans, agent task plans, system test plans |
| [04_architecture](04_architecture/) | ADRs, architecture flows, region maps, design rules |
| [05_design](05_design/) | Detailed design docs, phase files, subsystem designs |
| [06_spec](06_spec/) | Generated specs, spec categories, runtime specs, app specs |
| [07_guide](07_guide/) | User and developer guides: build, CLI, language, FFI, platform, security, API reference |
| [08_tracking](08_tracking/) | Bug tracking, audit logs, baselines, benchmarks, test results, agent logs |
| [09_report](09_report/) | Completion reports, status reports, yearly summaries |
| [10_metrics](10_metrics/) | Coverage dashboards, build metrics |
| [11_archive](11_archive/) | Historical/superseded docs — read-only reference |

## System-Data Directories (not wiki)

| Directory | Owner | Notes |
|-----------|-------|-------|
| `test/` | Test runner (`bin/simple test`) | `test_db.sdn`, `test_db_runs.sdn` — do not move |
| `todo/` | TODO scanner (`bin/simple todo-scan`) | `todo_db.sdn` — do not move |
| `TODO.md` | Generated by `bin/simple todo-gen` | Rebuilt from `todo/todo_db.sdn` |

## Source Tool Path Notes

- `sspec_docgen` default output: `doc/06_spec` (requires bootstrap rebuild after recent path update)
- `traceability` generated spec root: `doc/06_spec`

## Maintenance

- New docs go into the most specific numbered folder.
- Completed/superseded work moves to `11_archive/` rather than being deleted.
- `08_tracking/` holds all living bug/audit/baseline tracking; `09_report/` holds closed-out write-ups.
- Do not add ad-hoc unnumbered top-level folders — pick a number or use `11_archive/`.
