# Local Research: Simple Theme System

## 1. Current State of Theming

### Data Structures
- `UITheme`: Base theme class with `ThemeColors`, `Spacing`, `BorderStyle`, and `FontStyle`.
- `GlassDesignTokens` / `GlassColorTokens`: Extended tokens for glassmorphism effects (blur, translucency).
- `IOSDesignTokens`: Extended tokens for iOS-like styling.

### Renderers
- **Web/HTML**: Consumes CSS variables generated by `theme_to_css(UITheme)` and `glass_tokens_to_css(GlassDesignTokens)`.
- **Baremetal/WM**: Consumes numeric constants from `glass_numeric_tokens.spl`.

### Synchronization (Stitch Round-trip)
- `src/app/cli/theme_sync.spl`: CLI tool for dumping/diffing local tokens against Stitch SDN snapshots.
- `src/lib/common/ui/theme_sync_sdn.spl`: Handles serialization of `StitchDesignSystem` to/from SDN.

## 2. Gaps for "Simple Theme"

1. **CSS-to-Token Parsing**: No existing logic to parse a CSS file and populate `UITheme` or `GlassDesignTokens`.
2. **Unified Setup File**: Themes are currently hardcoded in factory functions (e.g., `GlassDesignTokens.obsidian_dark()`).
3. **Round-trip Complexity**: Round-tripping with Stitch requires multiple steps (dump -> diff -> push).

## 3. Proposal for "Simple Theme"

### Setup Format: `.simple-theme`
A hybrid format that allows CSS-like variable definitions but supports SDN metadata for Stitch integration.

```css
/* @stitch project: 12496218458601315145 */
:root {
  --bg: #12121f;
  --accent: #c6c6c8;
  --glass-blur: 20px;
}
```

### Unification Strategy
1. **SimpleTheme Class**: A new class that composes `UITheme`, `GlassDesignTokens`, and `StitchMetadata`.
2. **CSS Parser for Simple Theme**: A lightweight parser to extract variables from `.simple-theme` files.
3. **WM Integration**: Update the Window Manager to load `.simple-theme` files at boot (via VFS).
4. **Stitch Round-trip**: Automate the conversion between `.simple-theme` CSS and Stitch SDN payloads.
