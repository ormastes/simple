# Internationalization (i18n) Guide

**Simple** supports localized error messages in multiple languages, making it easier to understand compiler errors in your preferred language.

## Supported Languages

- **English (en)** - Default
- **Korean (ko)** - Full support

## Quick Start

### Using the `--lang` Flag

The easiest way to see error messages in your preferred language is to use the `--lang` flag:

```bash
# English (default)
simple build main.spl

# Korean
simple build main.spl --lang ko
```

### Example

**Source file** (`test.spl`):
```simple
fn main():
    let x = 42abc  # Invalid number literal
```

**English output**:
```
error[E0004]: invalid integer literal: 42abc
  --> test.spl:2:13
   |
 2 |     let x = 42abc
   |             ^^^^^ invalid integer literal
```

**Korean output**:
```
오류[E0004]: 잘못된 정수 리터럴입니다: 42abc
  --> test.spl:2:13
   |
 2 |     let x = 42abc
   |             ^^^^^ 잘못된 정수 리터럴
```
