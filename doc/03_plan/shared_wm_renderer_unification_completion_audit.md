# Shared WM Renderer Unification — Completion Audit

Date: 2026-05-29

## Scope

This audit maps the active objective to current evidence. It is intentionally stricter than the current implementation state: uncertain or partial evidence is treated as incomplete.

## Requirement Matrix

| Objective requirement | Evidence needed for completion | Current evidence | Status |
|---|---|---|---|
| Electron, Tauri, and pure Simple web renderer share the same web render API interface | A single Simple API/trait/module imported by all three adapters, with tests proving the same request/response contract for HTML/snapshot/patch/input/capabilities/host-window operations and optional pixel output | `src/lib/common/ui/web_render_api.spl` exists and `ui.web`, `ui.electron`, and `ui.tauri` import it. `test/unit/app/ui/web_render_backend_api_spec.spl` covers those backends plus pure Simple browser artifact participation and now proves Electron/Tauri standalone and async render IPC are built from common `WebRenderRequest` artifacts. `src/app/ui.ipc/async_handler.spl` preserves complete common render IPC JSON and wraps legacy raw HTML only for compatibility. Host window command JSON delegates to `WebRenderHostWindowCommand`; snapshot, patch, and input wire payloads are represented by `WebRenderSnapshotEnvelope`, `WebRenderPatchEnvelope`, and `WebRenderInputEnvelope`, covered by `test/unit/lib/common/ui/web_render_api_spec.spl`. Web, Electron, and Tauri expose adapter helpers for common snapshot, patch, and input envelopes; the live Web runtime sends snapshot and patch WebSocket payloads through the common envelopes, and the browser boundary unwraps them for the retained renderer. Electron and Tauri snapshot/patch transport helpers now have behavioral equality coverage against the shared web backend helper output. The legacy WebSocket render push in `src/app/ui.web/server.spl` now derives its body from `WebRenderRequest`/`web_render_body_html` and carries the `simple_web` target while preserving browser-compatible `type:"render"` frames. Pure Simple browser event queueing records the common input envelope. `src/app/ui.web/render_cache.spl` adds persistent static-shell HTML caching keyed from the shared web render cache key while bypassing dynamic islands. The surface registry now has a Tauri kind and focused Tauri binding coverage. Remaining work is broader end-to-end live host WebView transport verification and true binary artifact storage, not bespoke envelope construction. | Partially complete |
| CUDA, Metal, and CPU 2D renderers share the same 2D rendering API interface | CPU, Metal, and CUDA backend files all implement `std.gpu.engine2d.backend.RenderBackend`; backend selection compiles; tests assert mandatory method parity and capability handling | `RenderBackend` exists in `src/lib/gc_async_mut/gpu/engine2d/backend.spl`; CPU, Metal, and CUDA concrete files exist. CUDA now accepts an explicit `CudaSession`, includes PTX source and launch/readback wiring for `kernel_clear`, `kernel_draw_rect_filled`, and `kernel_draw_rect_outline`, reports typed context/kernel diagnostics, and only reports initialized after a hardware readback self-test proves clear, filled-rect, and outline pixels. Strict CUDA integration coverage proves `draw_rect` writes only outline pixels. Broader CUDA 2D operation coverage still mirrors through software or remains unimplemented beyond that scoped parity. | Partially complete |
| Host WM and SimpleOS WM share logic | Host and SimpleOS import/use the same WM service/core for create/focus/move/resize/minimize/restore/close/input routing; no local host service shim duplicates WM authority | `host_compositor_entry.spl` imports the real `os.services.wm.wm_service.WmService` for lifecycle state and translates host bridge requests into shared `WmAction` values before applying compositor mutations. Pointer-driven drag/resize, host direct focus, and host direct destroy now route through shared `WmAction` lifecycle helpers. `DesktopShell.apply_wm_action` now routes create, create-web, destroy, focus, resize, move, set-title, minimize, maximize, restore, and update-tree compositor mutation through `apply_wm_action_to_compositor(...)` while keeping shell-specific owner registration, launcher accounting, replies, and logs local. Shared update-tree materialization now lives in `wm_action_remote_tree(...)` instead of the desktop shell. Host still has a local `HostCompositor` adapter for host-native surfaces/input policy. | Partially complete |
| Host WM and SimpleOS WM use the same web renderer API | Host and SimpleOS web rendering path both call shared web render API, with tests for the same fixture | `simple_web_window_renderer.spl` builds `WebRenderRequest` values and turns Simple Web pixels into `WebRenderArtifact` values before blitting to `CompositorBackend`. Host compositor renderer-name exposure uses `WEB_RENDER_TARGET_SIMPLE_WEB`. Tests cover artifact dimensions, target identity, and host renderer listing. Host still has host-native surface adapter code outside the common web API. | Partially complete |
| Host WM and SimpleOS WM use the same 2D engine API | Both paths render through `CompositorBackend`/`Engine2D RenderBackend` adapters and do not bypass with direct platform-specific overlay drawing | Compositor/Engine2D adapters exist. The SimpleOS baremetal overlay painter now targets `CompositorBackend` and `DesktopShell.run_baremetal` wraps its Engine2D instance in `BaremetalEngine2dOverlayBackend`, with `test/unit/os/desktop/shell_baremetal_backend_spec.spl` proving the painter no longer requires raw Engine2D. This is still not full compositor-surface sharing: the baremetal overlay remains a shell-owned immediate-mode overlay because WidgetNode DSL/surface materialization is documented as unsafe in baremetal. | Partially complete |
| Pure Simple GUI/web-render path is memory and size optimized against an equivalent Qt GUI app | A measured Simple artifact and a measured equivalent Qt artifact, with the Simple path at or below Qt size and no avoidable allocation regressions in the pure Simple renderer | `src/app/ui.browser/backend.spl` uses pre-sized framebuffer/pixel arrays and now builds P3 snapshot text through a pre-sized part array instead of per-pixel array growth. `src/app/ui.browser/renderer.spl` also initializes its legacy framebuffer with a pre-sized array literal instead of repeated `push`. `test/unit/app/ui/web_render_backend_api_spec.spl` covers the optimized snapshot text path. `scripts/check-qt-gui-size-baseline.shs` records the smallest measured Simple Web artifact from `doc/09_report/web_baremetal_size_audit_2026-05-28.md`; current value is 14,336 bytes for `Simple web placeholder URL facade`. `scripts/check-gtk-gui-size-speed-baseline.shs` now records a successful lightweight Simple HTML render-loop baseline in `doc/09_report/gtk_gui_size_speed_baseline_2026-05-29.md`; GTK compiled to a 14,472-byte minimal binary but speed is unavailable on this headless host. Qt development files are still missing on this host, so Qt equal-or-better remains unproven. | Partially complete |
| Research local with agents | Independent local explorer results for web renderer, 2D renderer, WM sharing, and artifact presence | Agents completed and findings were merged into `doc/01_research/local/shared_wm_renderer_unification.md`. | Complete |
