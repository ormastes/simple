# Simple Web Browser Production Hardening Detail Design

<!-- codex-design -->

## Scope

Selected scope is Feature Option C and NFR Option C from:

- `doc/02_requirements/feature/simple_web_browser_production_hardening.md`
- `doc/02_requirements/nfr/simple_web_browser_production_hardening.md`

The design hardens Simple Web as a production browser boundary and keeps the
rendering evidence gate release-blocking for local host capabilities.

## Modules

- `src/app/ui.web/session_token.spl`: HMAC session token issue/verify and
  production secret policy.
- `src/app/ui.web/origin_guard.spl`: allowed-origin checks before token minting,
  bearer verification, sensitive data return, or WebSocket upgrade.
- `src/app/ui.web/auth_params.spl`: shared bounded request parsing, body/head
  limits, request-id sanitization, JSON security headers, and HTML document
  security headers.
- `src/app/ui.web/server.spl`: normal TCP HTTP/WebSocket boundary, shared-WM
  launch behavior, fixed-window login burst state, live root response,
  `/wm/native_window` HTML surface, static script surfaces, and unknown-route
  JSON fallback.
- `src/app/ui.web/async_server.spl`: async HTTP/WebSocket boundary using the
  shared auth/body/head/header helper policy.
- `src/app/ui.web/tls_serve_loop.spl`: TLS serving path with no insecure secret
  fallback and shared response/header policy.
- `src/app/ui.web/ui_routes.spl`: authenticated `/ui/resume` and sensitive
  `/api/*` route policy.
- `src/app/ui.web/ws_handler.spl`: WebSocket upgrade method/path/key/header and
  frame-size policy.
- `src/app/ui.web/html.spl`, `src/app/ui.web/wm.js`, and
  `src/app/ui.web/retained_renderer.js`: generated/static browser clients that
  use WebSocket subprotocol bearer tokens, not query-string bearer URLs, and are
  served with cache/no-sniff script response headers.

## Algorithms And Error Handling

- Server startup loads `SIMPLE_UI_WEB_TOKEN_SECRET`; missing production secrets
  fail closed unless non-TLS local development explicitly sets
  `SIMPLE_UI_WEB_ALLOW_INSECURE_DEV_SECRET=1`.
- TLS startup never accepts the insecure development fallback.
- `/ui/login` first validates origin, request framing, and unauthenticated body
  limits, then applies the fixed-window login burst gate before issuing a token.
- `/ui/ws`, `/ui/resume`, and sensitive `/api/*` routes require allowed origin
  plus an origin-bound bearer before upgrade, resume, or state return.
- Legacy `/ws` and arbitrary upgrade-shaped routes are hidden before upgrade.
- Query-string bearer compatibility is non-authorizing even when the deprecated
  environment flag is present.
- Request heads, request lines, header lines, unauthenticated bodies, resume
  bodies, and WebSocket frame payloads are bounded before large reads or
  allocations.
- Authenticated successful `/api/state` and `/api/widgets` `200 OK` JSON
  responses, JSON error responses, unknown-route fallback responses, and static
  script responses include no-store/no-cache/nosniff headers. HTML document
  responses, including the shown `/wm/native_window` surface, include nosniff,
  frame-deny, no-referrer, permissions policy, and restrictive CSP headers.
- `X-Request-Id` echoes only sanitized correlation IDs and rejects bearer-like
  values.

## Remaining External Gates

Local Linux evidence cannot prove every native host path. Keep these rows open
until verified on the matching host:

- macOS Metal native readback.
- AMD ROCm/HIP native readback.
- Windows DirectX native proof.
- Real browser WebGPU device readback.

Do not mark the feature tracking row `done` until those external host proofs are
available with accepted manifests captured through
`doc/09_report/simple_web_browser_external_host_evidence_manifest_template.md`
or the requirements are explicitly re-scoped.
