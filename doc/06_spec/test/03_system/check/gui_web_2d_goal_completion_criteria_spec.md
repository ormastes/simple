## Completion Rule

The overall goal is not complete until every scenario in this spec has
evidence-backed assertions over current evidence and passes. Passing the Linux
Vulkan row, retained perf lane, narrow renderer slices, or non-RenderDoc
browser Vulkan rows does not complete the goal by itself.

Until the SSpec runner failure-summary bug is fixed, automation must also run
`scripts/check/check-gui-web-2d-completion-criteria-placeholders.shs`. The
script fails while any `TODO(gui-web-2d-completion)` placeholder remains and
emits `gui_web_2d_completion_criteria_todo_count` evidence, so a file-level
runner `PASS` cannot be mistaken for broad GUI/Web/2D completion.

## Syntax

```sh
SIMPLE_LIB=src bin/simple test test/03_system/check/gui_web_2d_goal_completion_criteria_spec.spl --mode=interpreter --clean --fail-fast
```

The command is expected to fail while any platform or parity lane is incomplete.
That failure is intentional completion evidence: the first failing assertion
names the exact field that does not yet prove the goal.

## Evidence Sources

This spec uses the top-level aggregate audit instead of launching Chrome,
Electron, RenderDoc, Xcode, or PIX directly. The aggregate is generated by:

```sh
BUILD_DIR=build/test-gui-web-2d-goal-<lane>/out \
REPORT_PATH=build/test-gui-web-2d-goal-<lane>/report.md \
sh scripts/check/check-gui-renderdoc-feature-coverage-status.shs
```

Platform wrappers remain the source of truth for live native artifacts. The
aggregate only consumes their current evidence files and re-emits normalized
fields for completion assertions.

## Linux Vulkan Assertions

The Linux Vulkan lane is now evidence-backed. It requires:

- `linux_vulkan_render_log_compare_status=pass`
- `linux_vulkan_render_log_compare_required_api=vulkan`
- `linux_vulkan_render_log_compare_pairwise_status=pass`
- `gui_web_2d_vulkan_browser_backing_status=pass`
- `gui_web_2d_vulkan_simple_backend_status=pass`
- `gui_web_2d_vulkan_pixel_comparison_status=pass`
- all Electron/Chrome/Simple pairwise diff statuses `pass`
- Simple, Chrome, and Electron RenderDoc env files `pass`
- Simple, Chrome, and Electron RenderDoc artifact files `pass`
- Simple, Chrome, and Electron RenderDoc artifact magic `RDOC`

Host-tool readiness fields are useful diagnostics, but they do not satisfy the
completion proof unless the native `.rdc` artifacts also pass file-status and
magic checks.

## Remaining Placeholders

The macOS Metal, Windows D3D12, full HTML/CSS inventory, production GUI/Web
parity, and parallel-agent review lanes still use explicit
`TODO(gui-web-2d-completion)` fail-fast assertions. Replacing one of those
placeholders must add field-level assertions over the current aggregate or
platform wrapper evidence and update
`gui_web_2d_completion_criteria_placeholder_audit_spec.spl`.

## Acceptance

- Linux Vulkan completion proves Chrome, Electron, and Simple Vulkan backing,
  pairwise nonblank ARGB equivalence, and valid RenderDoc `.rdc` artifacts with
  `RDOC` magic for all required lanes.
- macOS Metal completion proves native Metal readback, browser/gui backing,
  pairwise equivalence, and Xcode GPU Capture evidence when capture is required.
- Windows D3D12 completion proves native D3D12/DXGI readback, browser/gui
  backing, pairwise equivalence, verified PIX artifact files, and verified GPU
  debugger artifact files.
- 4K and 8K showcase completion proves current-source retained rows at 200 FPS
  with timing, RSS, checksum/readback, viewport, native binary provenance,
  retained mode, redraw count, and `fallback_state=none`.
- Full HTML/CSS completion proves all inventory CSS properties are rendered,
  not only the implemented Simple Web subset.
- Production GUI/Web parity completion proves same-frame backend readback,
  positive backend handles, matching checksums, and no CPU-mirror-only pass.
- Parallel-agent completion records Spark or fallback sidecar outputs plus
  normal/high-capability review before accepting broad findings or done marks.
