# RISC-V Linux RTL Dual-Arch Completion NFR

- NFR-RLD-001: Shared contracts must be the single source of truth for ABI, XLEN, machine identity, and Linux handoff.
- NFR-RLD-002: Default FPGA manifest generation must be deterministic across both architectures.
- NFR-RLD-003: Dual-arch contract tests must run locally without requiring FPGA hardware.
- NFR-RLD-004: Final acceptance must include the external RTL Linux smoke checks for both RV32 and RV64.
