# FR: JWT sign/verify requirements (HS256, RS256, ES256)

- **Date filed:** 2026-07-06
- **Status:** Covered by existing tests (added for traceability only — see
  `scripts/check/cert/check-req-traceability.shs`)

## Summary

`src/lib/common` provides JWT (RFC 7515) sign/verify helpers for HS256,
RS256, and ES256. The requirements below are the minimal set already
exercised by `test/01_unit/lib/common/jwt_spec.spl`.

## Requirements

- REQ-JWT-001: HS256 header base64url-encodes per RFC 7515 Appendix A.1.
- REQ-JWT-002: HS256 payload base64url-encodes per RFC 7515 Appendix A.1.
- REQ-JWT-003: HS256 signature matches the RFC 7515 Appendix A.1 test vector.
- REQ-JWT-004: the full compact HS256 JWT matches the RFC 7515 Appendix A.1
  test vector byte-for-byte.
- REQ-JWT-005: HS256 sign-then-verify round-trips successfully.
- REQ-JWT-006: HS256 verify rejects a wrong key.
- REQ-JWT-007: HS256 verify rejects a tampered payload.
- REQ-JWT-008: `jwt_sign_hs256` produces a 3-part compact JWT.
- REQ-JWT-009: RS256 sign-then-verify round-trips successfully.
- REQ-JWT-010: ES256 sign-then-verify round-trips successfully.
- REQ-JWT-011: HS256 compact JWTs contain no base64 padding.
- REQ-JWT-012: `jwt_verify_hs256` rejects a non-JWT string.
