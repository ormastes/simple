## Technical Details

### Catalog Structure
Each catalog follows consistent format:
- 10 features per category (except special cases)
- 12 metadata fields per feature
- Code examples in raw strings
- Dependency and reverse-dependency tracking
- Comprehensive descriptions

### Feature Metadata Schema
```simple
FeatureMetadata {
    id: Int,
    name: String,
    category: String,
    difficulty: Int (1-5),
    status: String ("Complete"/"In Progress"/"Planned"),
    impl_type: String ("Rust"/"Simple"/"Rust+Simple"),
    spec_ref: String,
    files: List[String],
    tests: List[String],
    description: String,
    code_examples: List[String],
    dependencies: List[Int],
    required_by: List[Int],
    notes: String
}
```
