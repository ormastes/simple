# Android Development Guide

How to build and test the Simple GUI on Android devices and emulators.
Two rendering paths exist: **Tauri WebView** (HTML/CSS in a native shell) and
**WASM native surface** (no-JS, `wasm32-simple-ui-android`). Both share the
responsive CSS and 44px touch-target contract.

## 1. Prerequisites

| Requirement | Minimum version | Notes |
|-------------|----------------|-------|
| Android SDK | API 33+ | `~/Library/Android/sdk` or `$ANDROID_HOME` |
| NDK | 27.x | `$NDK_HOME` — required for Tauri native builds |
| Android Emulator | any recent | `emulator -avd Pixel7` |
| Rust toolchain | stable | `rustup target add aarch64-linux-android` |
| Tauri CLI | 2.x | `cargo install tauri-cli` |
| `simple` binary | on `$PATH` | `scripts/setup/setup.shs` or `bin/simple` symlink |

## 2. Environment Setup

```sh
export ANDROID_HOME=~/Library/Android/sdk
export NDK_HOME="$ANDROID_HOME/ndk/27.0.12077973"
export PATH="$ANDROID_HOME/platform-tools:$ANDROID_HOME/emulator:$PATH"
```

On Linux without a local SDK, install via `sdkmanager`:
```sh
sdkmanager "platform-tools" "platforms;android-34" "emulator" "system-images;android-34;google_apis;x86_64"
avdmanager create avd -n Pixel7 -k "system-images;android-34;google_apis;x86_64" --device "pixel_7"
```

## 3. Tauri WebView Path

Build, boot emulator, install, launch:

```sh
cd tools/tauri-shell
cargo tauri android build --debug --target aarch64   # → app-universal-debug.apk

emulator -avd Pixel7 -no-snapshot -gpu swiftshader_indirect &
adb install -r src-tauri/gen/android/app/build/outputs/apk/universal/debug/app-universal-debug.apk
adb shell am start -n com.simple.ui/.MainActivity
adb exec-out screencap -p > /tmp/android.png
```

The WebView loads the embedded static page generated by `web_render_full_html`.
For live-server mode, set `SIMPLE_DASHBOARD_URL` to the host machine's IP.

### 3.1 Runtime packaging and Vulkan evidence

Build the Android arm64 `simple` runtime before rebuilding the APK:

```sh
cd src/compiler_rust
ANDROID_HOME=~/Library/Android/sdk \
NDK_HOME=~/Library/Android/sdk/ndk/27.0.12077973 \
CC_aarch64_linux_android=~/Library/Android/sdk/ndk/27.0.12077973/toolchains/llvm/prebuilt/darwin-x86_64/bin/aarch64-linux-android24-clang \
AR_aarch64_linux_android=~/Library/Android/sdk/ndk/27.0.12077973/toolchains/llvm/prebuilt/darwin-x86_64/bin/llvm-ar \
CARGO_TARGET_AARCH64_LINUX_ANDROID_LINKER=~/Library/Android/sdk/ndk/27.0.12077973/toolchains/llvm/prebuilt/darwin-x86_64/bin/aarch64-linux-android24-clang \
cargo build --release --target aarch64-linux-android -p simple-driver --bin simple
```

Then rebuild the APK with the runtime embedded:

```sh
sh tools/tauri-shell/scripts/build-ui-bundle.shs
cd tools/tauri-shell
SIMPLE_ANDROID_RUNTIME_AARCH64=$PWD/../../src/compiler_rust/target/aarch64-linux-android/release/simple \
cargo tauri android build --debug --target aarch64 --apk --ci
```

Verify live Android renderer evidence with:

```sh
PATH="$ANDROID_HOME/platform-tools:$ANDROID_HOME/emulator:$PATH" \
sh scripts/check/check-tauri-android-mobile-renderer-evidence.shs
```

The evidence script installs the APK, launches `com.simple.ui/.MainActivity`,
captures logcat and a screenshot, and fails closed on `F/DEBUG`, `Fatal signal`,
`VulkanManager`, `Headless UI completed`, and subprocess parse failures. By
default it starts a no-window emulator with `-gpu host`, leaves Android's HWUI
renderer unchanged, and reads `build/tauri_android_mobile_renderer/emulator.out`
as supporting Vulkan evidence. Override `ANDROID_RENDERER_HWUI_RENDERER`,
`ANDROID_EMULATOR_GPU_MODE`, or `ANDROID_RENDERER_GPU_LOG` when testing another
device/emulator lane.

The wrapper persists normalized MDI proof rows in
`build/tauri_android_mobile_renderer/android_mdi_proof.validation.env` and
re-emits the validator-derived event, capture, performance, input-to-paint, and
animation rows. `android_mdi_interaction_latency_status=pass` and
`android_mdi_input_to_paint_ms>0` are required by the mobile parity aggregate.

On 2026-06-26 the arm64 runtime, APK packaging, and host Vulkan emulator lane
passed:
- `status=pass`
- `android_render_log_status=pass`
- `android_layout_status=pass`
- `android_vulkan_log_status=pass`
- screenshot:
  `/Users/ormastes/simple/build/tauri_android_mobile_renderer/android_renderer.png`
- logcat:
  `/Users/ormastes/simple/build/tauri_android_mobile_renderer/android_renderer_logcat.txt`
- GPU log:
  `/Users/ormastes/simple/build/tauri_android_mobile_renderer/emulator.out`

Known local emulator caveats:
- `debug.hwui.renderer=skiavk` crashes `libhwui`/`VulkanManager`.
- `-gpu swiftshader` and `-gpu lavapipe` still crash the guest VulkanManager.
- `-gpu swangle` exposes an ANGLE/Vulkan graphics adapter but the emulator died
  during APK install.
- `-gpu host` selects Apple M4 Vulkan in the emulator log; Android WebView's
  `dumpsys gfxinfo` may still report `Pipeline=Skia (OpenGL)`, so record this
  as host/emulator Vulkan translation evidence rather than guest HWUI `skiavk`
  evidence.

## 4. WASM Native Surface Path

The WASM path uses `WEB_RENDER_TARGET_ANDROID_WASM` (`"android_wasm"`) with a
no-JavaScript runtime. The target triple is `wasm32-simple-ui-android`.

### 4.1 Render contract

| Property | Value |
|----------|-------|
| Viewport | 412 x 915 (Pixel 7 logical) |
| Min touch target | 44px |
| Min input height | 44px |
| DPI | target-native |
| Layout | column-gap-8-padding-16 |

### 4.2 Hello GUI artifact

The deterministic hello-world GUI validates the rendering pipeline without a
full compiler:

```sh
bin/simple run src/lib/common/ui/wasm_hello_gui.spl
```

Key evidence functions:
- `wasm_hello_gui_artifact_for_target(WEB_RENDER_TARGET_ANDROID_WASM)` — HTML/CSS artifact
- `wasm_hello_gui_compile_plan_for_target(...)` — compile plan with linker, sysroot, output path
- `wasm_hello_gui_event_evidence(...)` — button/scroll/text event contract verification
- `wasm_hello_gui_surface_evidence_for_target(...)` — surface presence checks
- `wasm_hello_gui_target_package_for_target(...)` — full target package summary

### 4.3 Module contract

WASM modules must only import from the `simple_ui.*` namespace:
- `env.memory`, `simple_ui.present`, `simple_ui.input`, `simple_ui.image`,
  `simple_ui.text`, `simple_ui.time_now_ms`

Denied imports (filesystem, network, etc.) are rejected at validation time.

Required exports: `simple_app_init`, `simple_app_render`, `simple_app_event`.

## 5. Responsive Layout

The renderer emits responsive CSS keyed to `LayoutProfile` breakpoints from
`common/ui/profile.spl`:

| Size class | Width | Layout |
|-----------|-------|--------|
| compact | <= 600px | single column, full-bleed, 16px font, 44px touch |
| regular | 601–1200px | 2-column grid, 40px min controls |
| expanded | > 1200px | multi-column desktop |

Android phones typically hit the **compact** class. The WebView applies the
correct class from its real device width — no server-side plumbing needed.

## 6. Themes

No Android-specific theme exists yet. The default `dark` / `light` themes with
responsive CSS provide the baseline. A Material Design theme
(`android_light` / `android_dark`) with Material You tokens is a follow-up:
see `src/lib/common/ui/ios/` for the iOS equivalent to model against.

## 7. Testing on Linux (No Device)

Without a physical device or macOS-hosted emulator:

1. **Contract-level tests** verify generated HTML/CSS:
   ```sh
   bin/simple test test/01_unit/app/ui/web_wm_modern_shell_spec.spl
   ```
2. **WASM artifact evidence** checks module structure:
   ```sh
   bin/simple run src/lib/common/ui/wasm_hello_gui.spl
   ```
3. **Responsive preview** — open the generated preview HTML in a desktop browser
   with DevTools mobile emulation (Pixel 7: 412x915, touch simulation).

## 8. Troubleshooting

| Symptom | Cause / fix |
|---------|-------------|
| Page tiny / unreadable | Missing `width=device-width` viewport meta — ensure page came from current renderer. |
| Android emulator System-UI ANR | CPU starvation from concurrent builds — not a code bug. |
| "no space left on device" | Mobile builds + emulator need headroom; clean `src/compiler_rust/target/debug`. |
| "no bundled Simple runtime" card | Expected without cross-compiled `simple`; shell shows embedded static page. Use `SIMPLE_BIN` or `--url` for live subprocess. |
| WASM module rejected | Check imports against allowed list — filesystem/network imports are denied. |
