# API Surface Guard — AC-8

Part of the **perf-umbrella** (`perf-opt-lang-web-db-os`).  Ensures no
public symbol is silently added, removed, or renamed during performance
optimizations.

## Files

| File | Purpose |
|------|---------|
| `baseline.sdn` | Captured public-symbol baseline (303 symbols, 11 module files) |
| `quarantine.md` | Pre-existing broken specs excluded from regression checks |
| `arch_doc_hashes.txt` | SHA-256 hashes of canonical arch docs (generated by `--update-baseline`) |
| `README.md` | This file |

## How the guard works

1. **Snapshot** (`src/app/cli/api_surface_snapshot.spl`): emits a deterministic
   SDN listing every exported name from each public entry module.
2. **Diff** (`scripts/check/check-api-arch-guard.shs`): regenerates the
   snapshot into a scratch path, diffs it against `baseline.sdn`, and exits
   non-zero on any public-symbol delta (added/removed/renamed = RED).
3. **Arch-doc parity**: the guard also sha256-checks a curated list of
   high-value architecture docs.  A hash change = RED.

Run after every perf sub-goal:

```sh
sh scripts/check/check-api-arch-guard.shs
```

To refresh the baseline after a deliberate, reviewed API change:

```sh
sh scripts/check/check-api-arch-guard.shs --update-baseline
```

## Why grep over export — not symbol_analyzer / metadata_symbol_surface

Two existing compiler tools were evaluated:

- **`src/compiler/90.tools/symbol_analyzer.spl`**: an AST *consumer*.  Its
  entry point `SymbolUsageAnalyzer.analyze(nodes: [AnalysisNode], ...)` requires
  a pre-built node list from the compiler frontend.  There is no standalone
  source-file entry point — it cannot enumerate a module on its own.

- **`src/compiler/99.loader/metadata_symbol_surface.spl`**: `MetadataSymbolIndex`
  has three module slots (`m0..m2`) and four symbol slots each.  `symbols_for(path)`
  returns an `i32` count — not names.  Unusable for a diffable surface.

Fallback (explicitly authorised in the task spec): grep `^export ` over `*.spl`
files in the module root directories.  This is simple, fast, and covers 100 %
of public names because the language mandates explicit `export` declarations
(E0410 enforces this at check time).

## Covered modules (baseline.sdn)

| Domain | Module root |
|--------|-------------|
| Lang / Compiler driver | `src/app/cli` |
| Web (HTTP) | `src/lib/nogc_sync_mut/http` |
| DB | `src/lib/nogc_sync_mut/database` |
| DB interface | `src/lib/nogc_sync_mut/simple_db_if` |
| OS — NVFS | `src/os/services/nvfs` |
| OS — NVFS core | `src/os/services/nvfs/core` |
| OS — NVFS driver | `src/os/services/nvfs/driver` |
| OS — NVFS POSIX | `src/os/services/nvfs/posix` |
| OS — kernel net | `src/os/kernel/net` |

Scan depth: **maxdepth 1** (only files directly in the module root, not
sub-packages).  Sub-packages are guarded indirectly through the re-export
layer in `__init__.spl`.

## Scope limits

- No method-level or accessor-forwarding signature capture (that's a separate
  sspec, per the research doc).
- No struct-field or method-body inspection — name-level only.
- Signatures are not joined; the guard detects name presence/absence.
