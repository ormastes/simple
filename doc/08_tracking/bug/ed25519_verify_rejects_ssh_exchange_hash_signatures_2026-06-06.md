# Ed25519 Verify Rejects SSH Exchange-Hash Signatures

**Filed:** 2026-06-06
**Status:** Resolved 2026-06-06
**Area:** `src/os/crypto/ed25519.spl`, SSH host-key tests

## Summary

`ed25519_sign(seed, pubkey, message)` produces 64-byte deterministic signatures for SSH 32-byte exchange hashes. Earlier verifier checks with `ed25519_verify(pubkey, message, sig)` rejected at least the synthetic `_test_exchange_hash()` signature generated by `ssh_sign_exchange_hash`.

Current evidence now shows the local verifier accepting SSH exchange-hash signatures and the RFC8032 vector lane passing.

`rt_ed25519_verify` also rejects the generated SSH signature in both observed argument orders:

- `rt_ed25519_verify(hash, pubkey, sig)`
- `rt_ed25519_verify(pubkey, hash, sig)`

## Reproduction

Use `test/01_unit/os/apps/sshd/ssh_kex_hostkey_matrix_spec.spl` helpers to sign `_test_exchange_hash()` with `ssh_sign_exchange_hash("ssh-ed25519", ...)`, extract the raw signature, and verify it with `ed25519_verify(pubkey, hash, raw_sig)`.

Expected: verifier accepts the signature.

Actual at filing time: verifier returned `false`.

Actual on 2026-06-06 after the pure Simple Ed25519 lane was repaired: verifier returns `true`.

Attempted regression assertion on 2026-06-06:

```simple
val hash = _test_exchange_hash()
val raw_sig = _decode_sig_blob(ssh_sign_exchange_hash("ssh-ed25519", host_keys, hash).unwrap()).1
expect(ed25519_verify(pubkey, hash, raw_sig)).to_equal(true)
```

Result:

- `bin/simple test test/01_unit/os/apps/sshd/ssh_kex_hostkey_matrix_spec.spl --clean --json`
- `total_passed: 28`, `total_failed: 1`
