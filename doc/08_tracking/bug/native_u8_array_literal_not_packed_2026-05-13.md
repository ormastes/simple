# Native `[u8]` array literal is not a packed byte buffer

Date: 2026-05-13
Status: Open

## Summary

Replacing CRC32's generated byte table with a direct `[u8]` literal passed
interpreter KATs but failed the native benchmark checksum. The native row
reported CRC32 checksum `8589934432` while the C and Rust references reported
`352`.

## Reproduction

Change `_ieee_table()` in `src/os/crypto/crc32.spl` from `_make_table(...)` to
a 1024-byte `[u8]` literal holding the canonical IEEE table, then run:

```bash
SIMPLE_BENCH_SAMPLES=3 SIMPLE_BIN=src/compiler_rust/target/release/simple SIMPLE_RUNTIME_PATH=src/compiler_rust/target/release SIMPLE_NATIVE_ALL_PATH=src/compiler_rust/target/release test/perf/port_algorithms/run_port_algorithm_benchmarks.shs
```

The run fails with:

```text
[algbench-error] checksum mismatch alg=crc32 c=352 rust=352 simple=8589934432
```

## Notes

- `src/compiler_rust/target/release/simple check src/os/crypto/crc32.spl ...`
  passes.
- Interpreter CRC32/CRC32C KATs pass.
- Native mode is therefore not preserving the packed-byte semantics needed by
  `rt_typed_bytes_u32_le_at` when the buffer comes from a direct `[u8]` literal.

## Impact

CRC32 cannot safely use a static literal table as a workaround for the native
global-cache crash. Keep the generated table path until native packed byte-array
literals or native module-level array caches are fixed.
