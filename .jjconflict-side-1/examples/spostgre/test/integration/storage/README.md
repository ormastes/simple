Integration tests for the spostgre storage layer exercising the WAL writer, page-map writer, and NVFS shim together. Tests include a remount simulation (extract arena byte-images, re-inject into fresh arenas, recover and look up) and a CRC-fence scenario proving that a corrupted WAL record stops `wal_recover_tail` at the correct boundary. The `std.fs_driver` RamFs driver is mounted at `/db` to represent the storage namespace that a real NVFS integration would own; the gap between the MountTable surface and the in-process shim is tracked as FR-STORAGE-E2E-001.
