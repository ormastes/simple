# ELF SYMTAB Weak Symbol Research: Boot-Aliases in SimpleOS Kernel

**Bug:** BUG-COMPILER-ELF-SYMTAB — Cranelift emitter places WEAK-binding symbols
in the LOCAL portion of the `.symtab`, violating the `sh_info` invariant.
`ld.lld` rejects the output. Roadmap item: `doc/03_plan/remaining_roadmap.md` §2.

---

## 1. What Is a "Boot-Alias" in This Codebase?

A **boot-alias** is a weak symbol in the boot/runtime stub object that the linker
is expected to override with a strong mangled symbol from compiled Simple code.

The mechanism (Rust side, `linker.rs:freestanding_weak_boot_defsyms`):

1. Scan all **boot objects** (C stubs compiled with `__attribute__((weak))`) for
   symbols whose `nm` type letter is `W`, `w`, `V`, or `v` (ELF STB_WEAK).
2. For each such weak symbol `raw`, find a **strong** Simple-compiled symbol that:
   - Is listed in `imports.all_mangled[raw]` (exact variant match), OR
   - Ends with the suffix `__<raw>` (suffix fallback, e.g.
     `kernel__abi__syscall_shim__spl_handle_enter_user_blocking` ends with
     `__spl_handle_enter_user_blocking`).
3. Pass `--defsym=<raw>=<strong>` to `ld.lld` (or `-Wl,--defsym=...` via cc).

The **bug**: when Cranelift emits these weak symbols into its own object file
(e.g. when Simple code itself defines a symbol that happens to be weak), the
`weak: symbol.is_weak()` field at `cranelift.rs:306` is read correctly from the
`object` crate but the resulting ELF `.symtab` entry lands in the LOCAL partition
(before `sh_info`) instead of the GLOBAL/WEAK partition. This breaks the
`sh_info` invariant (`sh_info` must equal the index of the first non-LOCAL sym).

## 3. How the Pure-Simple ELF Writer Determines Weakness

**File:** `src/compiler/70.backend/linker/elf_writer.spl`

The writer does NOT assign `STB_WEAK` itself — it relies entirely on the `st_info`
byte already embedded in each `ElfWriterSymbol` passed by the caller:

```
fn sort_symbols_locals_first(symbols: [ElfWriterSymbol]) -> [IndexedSymbol]:
    val binding = (sym.st_info >> 4) & 0xf   # extract STB from st_info
    if binding == STB_LOCAL:
        locals = locals.push(indexed)
    else:
        globals = globals.push(indexed)       # WEAK lands here, alongside GLOBAL
```

Constants: `STB_LOCAL=0`, `STB_GLOBAL=1`, `STB_WEAK=2`
(defined in `src/compiler/70.backend/linker/elf_parser.spl:36`).

The `sym_resolver.spl` (`elf_binding_to_sym`) correctly maps `STB_WEAK -> SymBinding.Weak`.

**The pure-Simple writer is correct**: it correctly places WEAK symbols in the
non-LOCAL partition (the `else` branch). The bug is upstream in the Cranelift
backend, which produces `st_info` bytes with STB_LOCAL for symbols that should
be STB_WEAK before the pure-Simple writer ever sees them.

## 4. Root Cause Location

**File:** `src/compiler_rust/compiler/src/codegen/cranelift.rs` around line 306.

The function is a Mach-O object copy/rewrite utility. `symbol.is_weak()` is
called correctly, but the resulting `write::Symbol { weak: true, scope: ... }`
is serialized to ELF with an incorrect `st_info` — the `scope` field dominates
over `weak` when the `object` crate writes ELF, or the Cranelift module itself
does not set the weak linkage flag before emitting.

The **`object_parser.rs` path** (line 111) correctly handles `is_weak()` for
the SMF/object-parse path — the bug is specific to the Cranelift direct-emit path.
